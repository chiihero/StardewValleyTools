# Stardew Valley Mod Manager 实施计划

## 目标

把当前工具升级为 **GUI Mod 库管理器 + 一键导入器 + 批量 AI 汉化助手**。

核心规则：

- 启用 / 停用只改管理器记录；
- 导入到游戏时，只复制启用的 Mod；
- 不自动删除游戏 `Mods` 目录里其他 Mod；
- 暂不做备份与回滚。

## 第一阶段：基础架构

- 增加本地状态文件；
- 增加设置页；
- 增加 Mod 记录模型；
- 增加导入任务与结果模型；
- 统一日志与后台任务处理。

## 第二阶段：设置页

必须支持：

- Mod 库目录；
- 游戏目录；
- 游戏 `Mods` 目录；
- AI 开关；
- API Key；
- 模型名；
- Base URL；
- AI 测试按钮；
- 汉化功能开关；
- 导入策略（覆盖 / 跳过 / 询问）。

## 第三阶段：Mod 管理页

必须支持：

- 扫描 Mod 库；
- 列表展示；
- 搜索 / 筛选 / 排序；
- 启用 / 停用记录；
- 备注 / 标签；
- 基础详情：名称、作者、版本、类型、路径。

## 第四阶段：导入功能

必须支持：

- 一键导入启用的 Mod；
- 递归复制目录；
- 导入进度；
- 导入结果汇总；
- 冲突提示；
- 失败跳过 / 重试。

## 第五阶段：AI 汉化

必须支持：

- 从 Mod 管理页对已启用 Mod 批量调用 AI；
- 扫描 `i18n`；
- 识别缺失键；
- 调用 AI 生成中文；
- 使用设置页里的 OpenAI 配置；
- 结果预览与保存。

## 暂不做

- 备份 / 回滚；
- 自动下载 / 自动更新；
- 软链接 / 硬链接；
- 多 Profile；
- 自动清理游戏目录。
